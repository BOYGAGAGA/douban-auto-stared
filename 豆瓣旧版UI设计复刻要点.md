# 豆瓣旧版UI设计复刻要点

## 豆瓣旧版 UI 设计复刻要点（用于网页开发）

以下是基于提供的 3 张页面截图总结的设计规范与必须实现的要点，确保复刻出一致的视觉与交互风格：

---

### 一、整体风格定位

- 极简克制的内容导向设计，优先突出社区内容本身，UI 装饰极少，不干扰内容阅读

- 经典 Web 2\.0 桌面端风格，朴素实用，阅读友好

- 所有正文、标题文字统一左对齐，保证长文本阅读舒适度

### 三、布局规范（必须实现）

1. **页面整体约束**

    - 主体内容最大宽度限制为 1000px，水平居中，大屏下两侧自动留白，避免行宽过宽影响阅读

    - 顶部的两个导航栏（全局顶栏、页面导航）为全屏通栏，不受最大宽度限制

2. **双栏布局（所有页面统一）**

    - 左侧主内容区：占 70% 宽度，承载核心内容（动态流、个人模块、内容列表）

    - 右侧侧边栏：占 30% 宽度，承载辅助信息（用户信息、推荐、热门话题、关注列表）

3. **间距规范**

    - 所有功能模块之间保留 20px 的统一留白，保证呼吸感

    - 内容项内的行高、间距统一，保证阅读舒适

### 四、核心组件规范（必须实现的标志性组件）

#### 1\. 双层顶部导航

- 第一层（全局顶栏）：最顶部通栏，左对齐排列各产品入口（读书 / 电影 / 音乐 / 播客 / 同城 / 小组 / 阅读 / FM / 时间 / 豆品），右对齐排列用户操作（下载豆瓣客户端、提醒、当前用户名）

- 第二层（页面导航）：全局顶栏下方通栏，左对齐：品牌 logo \+ 页面导航链接（首页 / 我的豆瓣 / 浏览发现 / 话题广场等），右对齐：圆角搜索输入框（带占位提示文字）\+ 放大镜搜索按钮

#### 2\. 豆瓣标志性模块标题

这是最具辨识度的设计，必须严格实现：

- 格式：`[模块名称] + 一串英文点(...) + [右侧操作/统计信息]`

- 示例：

    - `我的游戏 ······ (在玩1·想玩8·玩过39)`

    - `热门话题 ······ (去话题广场)`

    - `豆瓣时间 ······ (查看全部)`

- 样式：模块名称使用品牌色、加粗、字号比正文大 2px；右侧文字使用常规字号、灰色辅助色

#### 3\. 动态 / 内容列表项

- 结构：左侧小方形头像 / 封面图，右侧内容区

- 内容区结构：

    1. 顶部：用户名 / 内容标题（蓝色链接）

    2. 中间：正文内容，支持文字、图片（图片最大宽度不超过主内容区，按比例自动缩放）

    3. 底部：操作栏，浅灰小字，依次排列：发布时间、回应、赞 \(数量\)、转发

- 分隔：每个列表项之间用浅灰分割线分隔

#### 4\. 封面网格模块

- 用于个人主页的「我的游戏」「我读」等模块，多个内容封面图横向等宽排列，统一尺寸，仅展示缩略图，无下方文字

#### 5\. 侧边栏模块

- 每个辅助模块独立，标题同样使用上述的点分隔标准格式，内容为推荐卡片、话题列表、关注头像列表等辅助信息

#### 6\. 小型操作按钮

- 如「\+ 写日记」「\+ 发照片」这类按钮，样式为：浅背景、细灰色边框、小圆角、灰色文字，尺寸小巧

---

### 五、必须实现的细节

1. 所有用户头像均为**方形，无圆角**（符合截图中的旧版设计，非新版的圆形头像）

2. 五星评分组件：用于内容评分，5 个灰色星星，实心为已评分，空心为未评分

3. 所有输入框（搜索框、分享输入框）统一为圆角设计，占位文字为浅灰色

4. 长列表页顶部提供排序筛选栏，以文字链接形式提供排序、筛选选项

5. 长列表底部提供分页信息（如`1-15 / 973`）和翻页控件

6. 链接 hover 交互：蓝色链接默认无下划线，鼠标悬浮时显示下划线
