# 豆瓣自动标记软件 — 设计文档

## 概述

一款运行在PC上的桌面应用，用户输入文字或图片后，自动在豆瓣搜索对应项目并在资源站搜索可用资源，确认后自动完成豆瓣标记操作。

## 技术栈

| 层级 | 技术 |
|------|------|
| 桌面框架 | Electron |
| 前端 | HTML + CSS + 原生JS |
| OCR | Tesseract.js (本地) |
| AI视觉 | Claude Vision API |
| 浏览器控制 | Playwright (备用) |
| 豆瓣交互 | 复用Edge Cookie的HTTP请求 |
| 资源站搜索 | HTTP请求 + 页面解析 |

## 核心功能

### 1. 输入模块

**文字输入：** 输入框 + 回车触发搜索。

**图片输入（四种方式）：**
- 拖拽图片到窗口
- Ctrl+V 粘贴图片
- 点击选择本地文件
- 截图快捷键（框选屏幕区域）

**图片处理流程：**
1. Tesseract.js 本地OCR提取文字
2. 若OCR置信度 < 70%，调用 Claude Vision API 识别
3. 提取关键词用于搜索

### 2. 搜索模块

- 豆瓣搜索：关键词请求豆瓣搜索接口，解析返回条目列表
- 资源站搜索：同时请求 https://www.xn--wcv59z.com/ 搜索接口
- 两个搜索并行执行，结果各自推送到对应标签页

### 3. 结果展示模块

**标签页切换：**
- 标签A：豆瓣结果（封面缩略图 + 名称 + 评分 + 类型标签）
- 标签B：资源站结果（资源名称 + 类型 + 下载/观看链接）

支持：电影、电视剧、书籍、音乐、游戏全部类型。

### 4. 确认弹窗

点击豆瓣结果后弹出，显示：
- 封面大图
- 名称、类型、豆瓣评分（五星组件）
- 简介、导演/作者、标签
- 操作按钮：标记为"想看/想读/想听/想玩" 或 "看过/读过/听过/玩过"
- 取消按钮

### 5. 自动标记模块

- 读取Edge浏览器的豆瓣Cookie
- 用Cookie调用豆瓣对应API完成标记
- 每次操作随机延迟2-5秒
- 单日标记上限50次
- User-Agent使用真实Edge标识

## UI设计规范

严格复刻豆瓣旧版设计，详见 `豆瓣旧版UI设计复刻要点.md`。

**核心配色：**
- 顶栏背景 `#333333`，文字 `#ffffff`
- 导航背景 `#f0f5e9`（主站）/ `#e8f2f8`（电影）
- 品牌主色 `#218c38`（主站）/ `#2d78b5`（电影）
- 标题 `#333333`，正文 `#666666`，辅助 `#999999`
- 链接 `#2777bb`，分割线 `#eeeeee`

**布局：**
- 双层顶栏导航
- 主内容最大宽度1000px居中
- 标志性模块标题：`模块名称 ······ 右侧操作`
- 方形头像（无圆角）

## 防封策略

1. 复用Edge Cookie，请求等同正常浏览器行为
2. 随机延迟2-5秒
3. 单日标记≤50次
4. 真实Edge User-Agent

## 成功标准

1. 输入文字或图片后3秒内返回搜索结果
2. 图片识别准确率 > 80%（常见书籍封面、电影海报）
3. 豆瓣标记成功率 > 95%
4. UI视觉与豆瓣旧版高度一致
